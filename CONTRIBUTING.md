# Contributing

## Open an issue
- Use GitHub Issues to report bugs, request enhancements, or ask implementation questions.
- Include reproduction steps, expected behavior, and actual behavior where applicable.

## Keep pull requests small and focused
- Submit PRs with a single clear purpose.
- Avoid mixing unrelated refactors with behavior changes.

## Update docs with behavior changes
- When changing runtime behavior, update relevant docs (such as `README.md` and `docs/`) in the same PR.

## Update changelog for user-visible behavior changes
- If behavior visible to users changes, update `CHANGELOG.md` in the same PR.

## Do not describe unimplemented features
- Keep documentation and PR descriptions aligned with what current `main` actually implements.
