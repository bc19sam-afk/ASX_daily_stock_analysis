# AU/US（ASX + US）量化分析系统审查（2026-03-26）

> 目标：不是工程美化，而是提升“可交易性、可复盘性、可控风险”的量化价值。

## 1) 信号定义是否清晰、是否可执行

### 当前现状
- 系统有一套明确的技术面打分框架：趋势/乖离率/量能/支撑/MACD/RSI 六维打分，总分 100，并映射到买卖信号。  
- 同时，最终交易建议（`operation_advice`、`decision_type`）主要由 LLM 生成并解析，JSON 失败时会退化为关键词启发式。

### 问题
- 出现“双信号源”：规则引擎给出 `buy_signal/signal_score`，LLM 再给一套 `operation_advice`，两者缺少一致性约束。
- LLM 输出失败时会用关键词数粗糙兜底，可能把复杂情境压缩成“买/卖/持有”。

### 为什么重要
- 交易执行需要“可复现信号定义”，否则无法稳定评估策略边际。
- 当信号定义不可复现时，回测和实盘的闭环会断裂。

### 最小可落地改法
- 增加“信号主键”：以规则引擎 `buy_signal + signal_score` 作为主决策字段，LLM 仅负责解释层（原因、新闻、风险叙述）。
- 保存并展示“规则信号 vs LLM建议 一致性标志（agree/disagree）”。
- 当 JSON 解析失败时，直接降级为 `WAIT/HOLD`，禁止自动推导成激进买卖。

### 优先级
- **P0**

---

## 2) 现有输出是否能支持实际交易决策

### 当前现状
- API 输出包含摘要、操作建议、情绪评分、止损止盈等字段。
- Prompt 已要求给出 ATR 风险预算、建议股数、止损目标等交易要素。

### 问题
- 交易执行关键字段（建议仓位股数、风险金额、预期R倍数）并未形成强类型落库字段，依赖 LLM 文本/JSON 结构稳定性。
- 目前报告偏“分析可读性”，但缺“下单参数完整性检查”（入场、止损、目标、仓位、失效条件、持有期限）。

### 为什么重要
- 真正下单需要参数完整、结构稳定、可校验，而不是仅有叙述性建议。

### 最小可落地改法
- 在 `AnalysisHistory` 增加结构化字段：`position_size_shares / risk_amount / entry_valid_until / thesis_invalidated_if`。
- API 层增加 `execution_ready` 布尔值（必须同时满足：entry/stop/tp/size 全部有效）。

### 优先级
- **P0**

---

## 3) 是否缺少最小回测 / 复盘能力

### 当前现状
- 已有可运行的 long-only 回测引擎：可评估方向正确率、win/loss、止损止盈触发、模拟收益。
- Web 端也有回测页面，能看汇总指标和逐条结果。

### 问题
- 回测窗口单一（默认短窗），且核心是“信号评估”，不是“组合资金曲线回测”。
- 缺少按市场（ASX/US）、按 regime（震荡/趋势）分层统计；缺少 walk-forward 或滚动窗口稳定性验证。

### 为什么重要
- 只有单一窗口的命中率，容易过拟合或在不同市场状态失效。

### 最小可落地改法
- 增加三档固定窗口（5/10/20 交易日）并并行汇总。
- 增加最小“月度复盘表”：每月信号数、胜率、平均收益、最大回撤（模拟）。
- 增加按市场标签（ASX/US）拆分统计。

### 优先级
- **P1**

---

## 4) 是否缺少仓位管理 / 风险控制

### 当前现状
- Prompt 层已经写入“单笔亏损不超过总资金 1% + ATR 计算仓位”的理念。
- 回测端支持 stop_loss/take_profit 命中统计。

### 问题
- 风控约束主要停留在“提示词要求”，缺少后端硬约束与执行前校验。
- 没有组合层硬风控：总风险敞口、行业集中度、相关性上限、同向信号并发上限。

### 为什么重要
- 没有硬风控，策略再好也可能被单日相关性冲击破坏。

### 最小可落地改法
- 在下发建议前做 `pre_trade_risk_check`：
  - 单笔风险 ≤ 1%
  - 当日新增总风险 ≤ 3%
  - 单行业敞口 ≤ 35%
- 不通过则把建议自动降级为 `观望` 并给出拒绝原因。

### 优先级
- **P0**

---

## 5) 是否缺少观察池分层与候选排序

### 当前现状
- 有 ASX200 扫描器，能给出候选并按相对强度排序。
- 主配置仍是单层 `STOCK_LIST`，没有“核心池/战术池/观察池”分层模型。

### 问题
- 扫描结果与主交易池缺少闭环：候选如何入池、淘汰、降级没有规则状态机。
- US 与 ASX 混合时，没有市场分桶排序（可能导致不同时段/波动特征混排）。

### 为什么重要
- 分层与排序是把“发现机会”变成“可执行名单”的关键中间层。

### 最小可落地改法
- 定义三层池：
  - Tier-1 核心持仓池
  - Tier-2 候选建仓池
  - Tier-3 观察池
- 每日按规则自动迁移（升级/降级）并记录迁移原因。

### 优先级
- **P1**

## 6) 当前哪些模块是在“工程完善”，哪些模块才真正提升量化价值

### 当前现状
- 工程成熟度较高：配置中心、API schema、通知、多端 UI、任务调度、容错都较完整。
- 已具备量化价值基础模块：规则信号引擎、回测引擎、扫描器、历史记录。

### 问题
- 当前迭代重心偏“系统可用性与集成能力”，而非“策略边际提升速度”。
- 仍缺少：统一信号定义、组合级风险预算、分层池迁移机制、稳定性复盘体系。

### 为什么重要
- 工程稳定是必要条件，但不是收益来源；收益来源是“信号-执行-复盘”闭环效率。

### 最小可落地改法
- 研发任务按“量化价值增益”分层：
  - Q-Value（直接提升策略有效性）
  - Q-Support（支持分析和执行）
  - Eng-Hygiene（工程维护）
- 每个迭代至少 60% 容量投向 Q-Value。

### 优先级
- **P2**

## 最值得先做的 Top 5 升级路线图（按先后顺序）

1. **统一决策主线：规则信号主决策，LLM只做解释层（P0）**  
   先解决“同一标的两套建议”的不一致问题。

2. **把执行参数结构化并强校验（P0）**  
   入场/止损/止盈/仓位/失效条件齐全才允许 `execution_ready=true`。

3. **实现下单前硬风控网关（P0）**  
   单笔风险、总风险、行业集中度三道闸，不过闸就自动降级观望。

4. **建立最小组合级复盘（P1）**  
   在现有回测基础上增加 5/10/20 日窗口 + 月度曲线/回撤统计 + ASX/US 分桶。

5. **建立观察池分层状态机（P1）**  
   扫描器候选自动进入 Tier-3，满足条件升级 Tier-2/Tier-1，并记录迁移原因。
